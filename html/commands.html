Colors
These color commands return the color values that can be used by other EasyCRT, EasyGDI, and Windows commands.

const color: array[-1..15] of longint;

EasyGDI provides color constants that make choosing colors easier by providing a basic selection of colors. COLOR[x] can be used wherever color values must be specified. The color order is copied right out of QBasic.

x

 Color

-1

 transparent

0

 black

1

 blue

2

 green

3

 cyan

4

 red

5

 magenta

6

 brown

7

 white

8

 gray

9

 light blue

10

 light green

11

 light cyan

12

 light red

13

 light magenta

14

 yellow

15

 bright white

Example of use:

COLOR[4] {returns the color value for red}

 

function RGB(R: Byte; G: Byte; B: Byte): LongInt;

This function allows you to choose your own colors by specifing amounts of red (r), green (g), and blue (b). r, g, and b can range from 0 to 255.

function GetRValue(RGB: LongInt): Byte;
function GetGValue(RGB: LongInt): Byte;
function GetBValue(RGB: LongInt): Byte;

These functions do the opposite of what the RGB does. They can take a color value and return the amounts of red, green, or blue inside.

function gradient(color1,color2:longint; stepno,steps:integer):longint;

This function lets you input two colors (color1 and color2) and it will return a color that falls stepno / steps of the way between those two colors. You can use this function to "fade" between colors. 'steps' is the number of steps in your loop and 'stepno' is the current step number.

For example, if you wanted a circle that fades from blue to red with blue on the inside and red on the outside you could do this.

for radius:=1 to 100 do
  begin
    setpen(CRT,gradient(color[9],color[12],radius,100),solid,2);
    circle(CRT,100,100,radius,radius);
  end;

EasyCRT Color Text Commands
Use these commands to set the foreground and background colors of text in the EasyCRT window.

var foreground, background: longint;

These commands are pretty simple. To set the foreground (text) or color just say

foreground := SomeColorValue;
background := SomeOtherColorValue;

After you do this any text outputted on the CRT screen be colored with foreground and background colors that you specified.
Note that you cannot use the set the foreground color to COLOR[-1]

procedure setcolors(f,b:longint);

This command lets you set the foreground and background colors in a single statement. f is the foreground color and b is the background color.

procedure colorspot(x,y:integer);

This command will color the spot at X,Y with the current background and foreground colors;

procedure colorspots(x1,y1,x2,y2,shape:integer);

This command will color the spots in the shape defined by coordinates x1,y1,x2,y2 and shape with the current background and foreground colors.

shape

 Description

boxshape

 Fills the outline of a box with top left coordinates (x1,y1) and bottom right coordinates (x2,y2)

fillboxshape

 Fills the a box with top left coordinates (x1,y1) and bottom right coordinates (x2,y2)

Example:
colorspots(4,10,8,20,fillboxshape);  { fills box with foreground and background colors }

function getspotFcolor(x,y:integer):longint;
function getspotBcolor(x,y:integer):longint;

These commands return the foreground and background colors of a the character at x,y

EasyCRT Input Commands
These commands are ways of getting input from the user.

procedure resetkeys;

This function clears the character and virtual key buffers. These buffers can fill up if the user presses a key when your program is not receiving input. It is a good idea to include this command before using input functions such as read, readln, keypressed, or inkeyasc.

function inkey:word;

This function works in real time. If a key is being pressed then its virtual keycode will be returned, if no key is being pressed then it will return 0. This is useful when getting input from arrow keys in a game and in some other situations. This is not good for getting text input, since there is no key buffer and it will only return a key if it is being pressed at the same time the function is called.

You can see a listing of virtual keycodes in the Turbo Pascal help file, under the topic Virtual Key Codes


function inkeyasc:char;

This function is useful for getting text input. Whenever a key is pressed, the EasyCRT stores it in a buffer. This function returns the keys from this buffer in the order they were received. It will return a CHR(0) if no key has been pressed. Use the resetkeys to clear this buffer.

procedure showcursor;
procedure hidecursor;

These commands tell the EasyCRT window whether or not to display a blinking cursor in it's current position. This can be used to signify to the user that the program is accepting keyboard input. It works well with the inkeyasc function.

function mousex: integer;
function mousey: integer;

These commands retrieve the current x and y positions of the mouse.

var ldown,rdown: boolean;

These are two public EasyCRT variables. If the left button is down then LDOWN = TRUE otherwise LDOWN = FALSE. If the right button is down then RDOWN = TRUE otherwise RDOWN = FALSE.

procedure getclick;

This pauses the program until there is a mouse click. The location of the click is stored in the point variable, LASTCLICK. LASTCLICK.X is the x position of the mouse during the click and LASTCLICK.Y is the y position of the mouse during the click. LASTCLICK will store clicks of the left  mouse button whether used with getclick or not.

BMP Commands
EasyGDI allows you to draw to the screen, in memory or to files. Because there are so many different places to draw to, the drawing commands must be told where do to the drawing when they are called. This is the purpose of the BMP type variable. A BMP type variable stores information about a particular place to draw and can be thought of as a piece of paper or drawing surface. BMP variables can be created just like any other variable:

var x: BMP;

But in order to be useful, the BMP variable must be initialized. There are several initialization commands. Some of them tell the BMP to represent a particular area of the screen, others may point to a location in memory. The BMP variable CRT in EasyCRT is created automatically by the system, it represents the surface of the CRT Window.

When you initialize a BMP, you are giving it system resources. In order for those system resources to be freed it is very important to destroy the BMP when you are done using it. You do this using the killBMP function described below. The killBMP function frees the resources associated with the BMP. (Note: the CRT bitmap is automatically created and destroyed by the system.)

function isbmp(var it:BMP):boolean;

If you try to use a BMP variable in a command before it has been initialized or after it has been destroyed, the program will throw a run time error. This function returns true if the specified bitmap is invalid and can be used to prevent such errors.

function makewindowBMP(HWindow:hwnd):BMP;

Gets and encapsulated the device context associated with the window.

function makeblankBMP(CompatibleBMP:BMP; Width,Height:word):BMP;

Creates a new blank bitmap in memory of the specified size. A compatible BMP must be specified to gather information about the color depth and other attributes.

function LoadBMP(filename:string):BMP;

Loads a bitmap file from a disk into memory where it can be transfered and manipulated.

function encapsulateDC(DC:HDC):BMP;

Encapsulates an already existing device context with limited functionality. Useful for wm_paint message handling.

procedure killBMP(var it: BMP);

Frees resources associated with BMP.

procedure saveBMP(TheBMP:BMP; filename:string);

Saves a bmp into a file on a disk. Will not work on BMPs from encapsulateDC.

procedure drawpicture(TheBMP: BMP; filename: string; x,y:integer);

Quick method for loading a bitmap off a disk and copying it onto the screen. It is slow and should not be used for animations.

function DChandle(thebmp:BMP):HDC;

Returns the DC handle of a bitmap. A DC handles is used by most Windows GDI functions.Example: DChandle(CRT);

function getwidth(thebmp:bmp):integer;

Returns the width of a bitmap. Will not work on BMPs from encapsulateDC.

function getheight(thebmp:bmp):integer;

Returns the height of a bitmap. Will not work on BMPs from encapsulateDC.

procedure copybmp(Source,Destination:bmp; Dx,Dy:integer);

Copies the Source bitmap to point Dx,Dy on the Destination bitmap.

Can be used to display BMPs that are loaded from a disk.

Example: 

var b: BMP;
begin
  initwincrt;
  b := loadbmp('c:\russ\homer.bmp');
  copybmp(b,CRT,20,20);
  killBMP(b);
end.

procedure stretchcopy(Source,Destination:bmp; Dx,Dy,Dw,Dh:integer);

Does the same thing as copybmp except that Source is stretched to the size specified by Dw, Dh;

procedure piececopy(Source,Destination:bmp; Sx,Sy,Sw,Sh,Dx,Dy:integer);

Copies a piece of the the Source bitmap to point Dx,Dy on the Destination bitmap. The size and location of the piece are specified by the Sx,Sy,Sw,and,Sh paramaters

procedure maskcopy(Source,SourceMask,Destination:bmp; Dx,Dy:integer);

Does the same thing as copybmp but allows a bitmap mask to be specified for the source.

A mask is usually a greyscale bitmap the same size as the original bitmap. The area of the mask that are white are areas where the source will cover up the destination. The areas on the mask that are black will leave the destination colors intact. Gray areas of the mask will result in a combination of the source and destination colors. Greyed and colored masks can be used for special effects.

procedure supremecopy(Source, SourceMask, Destination: bmp;
                      Sx,Sy,Sw,Sh: integer;
                      Dx,Dy,Dw,Dh: integer);

The most flexible and direct way of copying from one bitmap to another. This function lets you mask, resize, and reposition the source all in one step. If you don't want to use a mask, just type in nil for Sourcemask. To use the default width and height of the source just type in 0 for Sw and Sh. To skip resizing, just use 0 for Dw and Dh.

Source and Destination must be filled in. SourceMask allows you to specify a mask for the source and can be nil if one does not exist. Sx, Sy, Dx, and Dy, must be filled in. Sw and Sh can be zero if you want to use the source's orginal height and width. Dw and Dh can be zero if you want them to equal the Sx and Sy.

EasyCRT Graphics Variables

var CRT:BMP;

This BMP variable points to the surface of the EasyCRT window. When you want to draw on the EasyCRT window using EasyGDI functions you specify CRT as the BMP variable;

For example:

circle(CRT,200,200,30,30);  {draws a circle on the CRT Window }

The CRT variable is initialized and deleted automatically by EasyCRT.

TheFont:FONT;

This variable stores the information about the font to be used in GDI text commands. It is automatically associated with the CRT window.

 

BMP Draw
These functions let you draw on BMP surfaces.

procedure setpen(TheBMP:BMP; color: longint; linestyle, width: integer);

Windows uses the pen style to draw lines, curves, and the borders of shapes. This function allows you to set the pen style of a particular BMP.

color - specifies the color of the pen
linestyle - specifies the style of the line

linestyle

 description

solid

 a solid line

dash

 a dashed line

dots

 a dotted line

dashdots

 a line with alternating dashes and dots
exdashdots a line with alternating dashes and double dots

width - specifies the thickness of the line. 0 is the default width. Specifying any width greater than 1 will cause the linestyle parameter to be ignored

Example:
setpen(CRT,color[2],dots,1);

procedure setbrush(TheBMP:BMP; color,bcolor:longint; style: integer);

Windows uses the brush style to draw fills and the insides of shapes. This function allows you to set the brush style of a particular BMP.

color - specifies the foreground color of the brush;
bcolor - specifies the background color of the brush;
style - specifies a brush style

linestyle

 description

solid

 solid colored

hlines

 horizontal lines

vlines

 vertical lines

ddlines

 downward diagonal lines
udlines upward diagonal lines
grid crossed lines
dgrid diagonal crossed lines

Example:
setbrush(CRT,color[12],color[-1],solid);

procedure box(TheBMP:BMP; x1,y1,x2,y2,x3,y3:integer);

This command draws a rectangle with corners (x1,y1) and (x2,y2). The rectangle is drawn with the current pen and filled with the current brush. The x3 and y3 tell how round to make the corners. If they are zero, the corners are perfectly straight, otherwise x3 specifies the width of the ellipses used to draw the rounded corners and y3 specifies their height.

procedure circle(TheBMP:BMP; cx,cy,radiusw,radiush:integer);

This draws a ellipse with center (cx,cy), horizontal radius RADIUSH, and vertical radius RADIUSV.


procedure pcircle(TheBMP:BMP; xpos,ypos,radiusw,radiush,angle1,angle2,way:integer);

This draws an arc from an ellipse with center (xpos,ypos), horizontal radius radiush, and vertical radius radiusv.

Angle1 and Angle2 are the starting and finishing angles of the arc in degrees going counterclockwise.

linestyle

 description
arcline Draws the arc only using the current pen.
arcslice Draws the arc as well as a line connecting the two endpoints with the current pen. It fills this shape with the current brush.
pieslice Draws the arc as well as lines connecting the two endpoints to the center of the ellipse with the current pen. Fills the shape with the current brush.


procedure line(TheBMP:BMP; x1,y1,x2,y2:integer);

This command draws a line from (x1,y1) to (x2,y2) with the current pen


procedure connectdots(TheBMP:BMP; var pointarray; count:integer);

If you want to draw lines in bulk, you can create an array of points and use a single command to connect them.

For example:
var myshape: array[1..5] of points;
begin
  initwincrt;
  myshape[1].x:=320; myshape[1].y:=200;
  myshape[2].x:=350; myshape[2].y:=250;
  myshape[3].x:=120; myshape[3].y:=300;
  myshape[4].x:= 34; myshape[4].y:=250;
  myshape[5].x:= 1; myshape[5].y:=200;
  connectdots(CRT,myshape,5);
end.

procedure shape(TheBMP:BMP; var pointarray; count,method: integer);

This function allows you to draw a polygon. The arguments are the same as for connectdots except for method described below.

 

method

 description
alternate When the polygon-filling mode is ALTERNATE, the system fills the area between odd-numbered and even-numbered polygon sides on each scan line. That is, the system fills the area between the first and second side, between the third and fourth side, and so on.
winding When the polygon-filling mode is WINDING, the system uses the direction in which a figure was drawn to determine whether to fill an area. Each line segment in a polygon is drawn in either a clockwise or a counterclockwise direction. Whenever an imaginary line drawn from an enclosed area to the outside of a figure passes through a clockwise line segment, a count is incremented (increased by one); when the line passes through a counterclockwise line segment, the count is decremented (decreased by one). The area is filled if the count is nonzero when the line reaches the outside of the figure.


procedure pset(TheBMP:BMP; x,y: integer; color:longint);

PSET plots a point at (XPOS,YPOS) with the specified color.


function pixel(TheBMP:BMP; x,y:integer):longint;

Pixel returns the color value of point (XPOS, YPOS)


procedure fill(TheBMP:BMP; x,y:integer; colorinfo:longint; filltype:integer);

Performs a flood fill starting at point (X,Y) with the current brush.

COLORINFO is not the fill color, its meaning depends on the value of FILLTYPE.

filltype

 description
border Fill area is bounded by the color specified by the COLORINFO parameter.
surface Fill area is defined by the color specified by the COLORINFO parameter. Filling continues outward in all directions as long as the color is encountered.

type FONT = record
  face: string;
  height,weight: integer;
  italic,underline,strikeout:byte;
  angle:real;
  fcolor,bgcolor:longint;
  halign,valign: word;
end;

The font type is record variable that is used to describe a font in EasyGDI. Here is an explanation of each of the fields:

field name

 description
face A string containing the font face (or one of the following font constants: times, arial, courier, cursive, comic, stencil, verdana, or ransom)
height The height of the font
weight A number from 0-1000 describing the weight of the font. 0 is default, 1 is lowest weight, 1000 is heighest weight;
italic Makes the font italic if not equal to 0
underline Makes the font underline if not equal to 0
strikeout Makes the font strikeout if not equal to 0
angle angle (in degrees) that specifies the rotation of  the font.
fcolor font color (cannot be color[-1])
bgcolor background color
halign Specifies horizontal alignment. Can be: ta_Left, ta_Right, or ta_Center
valign Specifies vertical alignment. Can be: ta_Top, ta_Bottom, or ta_BaseLine

procedure quickfont(var AFont:FONT; fface:string; fsize: integer);

This function will take a font variable and fill it with the face and size (height) that you specify. The rest of the values are set to defaults.


procedure setfont(ABMP:BMP; var AFont:FONT);

This function will associate a font with a BMP. Any text printed in that BMP will then have the characteristics specified in the font. For EasyCRT, the font variable, TheFont, is automatically associated with the CRT window.


procedure print(ABMP:BMP; x,y:integer; text:string);

This function prints text at coordinate (X,Y) using the current font.


procedure wrapprint(ABMP:BMP; x,y,w:integer; text:string);

This function prints wrapped text at coordinate (X,Y) using the current font. The line width is specified by W.


function getwrappedheight(ABMP:BMP; w:integer; text:string):integer;

This function returns the height of the text if it would be wrap printed at width W using the current font. This function does not take font angle into consideration.


function getwrappedwidth(ABMP:BMP; w:integer; text:string):integer;

This function returns the width  of the text if it would be wrap printed at width W using the current font. This function does not take font angle into consideration.


function gettextwidth(ABMP: BMP; text:string):word;

This function returns the width of the text if it would be printed using the current font. This function does not take font angle into consideration.


function gettextheight(ABMP: BMP; text:string):word;

This function returns the height of the text if it would be printed using the current font. This function does not take font angle into consideration.

EasyGDI Only Program Functions
These functions provide specific features for your programs. They require additional information such as window or instance handles in order to function. DO NOT USE THESE FUNCTIONS FOR EASYCRT PROGRAMS, there are separate versions of these commands which do not require instance handles under "EasyCRT Only Program Functions."

procedure wunfreeze(Wnd: Hwnd);

A program that is in a repeating loop is unable to process its messages. This causes it to stop responding to keypresses and mouseclicks and can cause the system to lock up. Putting an UNFREEZE in the loop forces the program to look for and respond to messages so nothing locks up.

procedure wdelay(Wnd:Hwnd; milliseconds:longint);

This command pauses the program for the specified number of milliseconds.

procedure wstartdelay(var t: longint);
procedure wfinishdelay(Wnd:Hwnd; t,milliseconds:longint);

function wFileOpen(HWindow:hwnd; path,ftype,extension: string):string;

This function opens the Open common dialog box. Path is the path with the default directory, ftype is a description of the file like 'Picture File' or 'Data File', and extension is a string containing the file extension you are looking for like 'JPG'. The function returns the path of the selected file.

function wFileSave(HWindow:hwnd; path,ftype,extension: string):string;

This function opens the Save As common dialog box. The sytax is the same as for FileOpen


function getapppath(instance: thandle): string;

This function returns the path of the program. Getapppath requires the instance handle of the application. This is in the variable HINSTANCE for all Pascal programs.


function getappdir(instance:thandle):string;

This function returns the directory of the program. This is useful for finding data files which are usually in the same directory as the program
