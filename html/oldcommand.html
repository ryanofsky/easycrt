Color Functions

The functions and constants return color values that can be used in many EasyCRT and EasyGDI commands.

const COLOR[X]: array[-1..15] of longint;

x

 Color

-1

 transparent

0

 black

1

 blue

2

 green

3

 cyan

4

 red

5

 magenta

6

 brown

7

 white

8

 gray

9

 light blue

10

 light green

11

 light cyan

12

 light red

13

 light magenta

14

 yellow

15

 bright white

The purpose of these color constants is to make choosing colors easier by providing a basic selection of colors. COLOR[x] can be used wherever color values must be specified.

function rgb(red,green,blue:integer):longint;
This function returns a color value from the RED, GREEN and BLUE values. These values range from 0-255.

function getrvalue(color:longint):byte;
function getgvalue(color:longint):byte;
function getbvalue(color:longint):byte;
These functions return the amount of red, green, or blue in a color value.

function gradient(color1,color2:longint; stepno,steps:integer):longint;
You can use this function to "fade" between colors. 'steps' is the number of steps in your loop and stepno is the current step number.

For example, if you wanted a circle that fades from blue to red with
blue on the inside and red on the outside you could do this.

for radius:=1 to 100 do
  begin
    setpen(CRT,gradient(color[9],color[12],radius,100),solid,2);
    circle(CRT,100,100,radius,radius);
  end;

Pen and Brush Functions
When windows draws a shape it draws lines and curves with the current pen and it fills shapes and areas with the current brush. The drawing commands in EasyCRT and EasyGDI also use pens and brushes. Here are the functions that are used to set the pen and brush in your program.

procedure setpen(color: longint; linestyle, width: integer);

The color value can be any value returned from a color function,
one of the COLOR[X] values, or -1. When the color is -1 the pen is transparent

Linestyle may be any of these values:
0 solid
1 dash
2 dots
3 dashes & dots
4 double dashes & dots

Width, is just the thickness of the pen, in pixels.

procedure Asetbrush(DC:SDC; color,bcolor:longint; style: integer);
procedure setbrush(color,bcolor:longint; style: integer);

The COLOR value can be any value returned from a color function, one of the COLOR[X] values, or -1. When the color is -1 the brush is transparent. When STYLE is from 1-6, COLOR refers to the color value of the lines and BCOLOR descibes the fill color in between the lines

Style can be any of these values
0 Solid color
1 Horizontal Lines
2 Vertical Lines
3 Downward Diagonal Lines
4 Upward Diagonal Lines
5 Crossed Lines
6 Diagonal Crossed Lines

Drawing Functions
These functions are what actually do the drawing. They use the pens and brushes specified using the setbrush and setpen commands.

procedure Aqline(DC: SDC; x1,y1,x2,y2:integer);
procedure qline(x1,y1,x2,y2:integer);

This command draws a line from (x1,y1) to (x2,y2) with the current pen.

procedure Aqcircle(DC:SDC; xpos,ypos,radiusw,radiush:integer);
procedure qcircle(xpos,ypos,radiusw,radiush:integer);

This draws a ellipse with center (XPOS,YPOS), horizontal radius RADIUSH, and vertical radius RADIUSV.

procedure Abox(DC:SDC; x1,y1,x2,y2,x3,y3:integer);
procedure box(x1,y1,x2,y2,x3,y3:integer);

This command draws a rectangle with corners (x1,y1) and (x2,y2). The rectangle is drawn with the current pen and filled with the current brush. The x3 and y3 tell how round to make the corners. If they are zero, the corners are perfectly straight, otherwise x3 specifies the width of the ellipses used to draw the rounded corners and y3 specifies their height.

procedure Aqarc(DC: SDC; xpos,ypos,radiusw,radiush,angle1,angle2,way:integer);
procedure qarc(xpos,ypos,radiusw,radiush,angle1,angle2,way:integer);

This draws an arc from an ellipse with center (xpos,ypos), horizontal radius radiush, and vertical radius radiusv.

Angle1 and Angle2 are the starting and finishing angles of the arc in degrees going counterclockwise.

Way tells windows how to draw the arc
0 Draws the arc only using the current pen.
1 Draws the arc as well as a line connecting the two endpoints with the current pen. It fills this shape with the current brush.
2 Draws the arc as well as lines connecting the two endpoints to the center of the ellipse with the current pen. Fills the shape with the current brush. Looks like a slice of pie.

procedure Apset(DC: SDC; xpos,ypos: integer; color:longint);
procedure pset(xpos,ypos: integer; color:longint);

PSET plots a point at (XPOS,YPOS) with the specified color.

function Apixel(DC: SDC; xpos,ypos:integer):longint;
function pixel(xpos,ypos:integer):longint;

Pixel returns the color value of point (XPOS, YPOS)

procedure Afill(DC: SDC; xpos,ypos:integer; colorinfo:longint; filltype:integer);
procedure fill(xpos,ypos:integer; colorinfo:longint; filltype:integer);

Performs a flood fill starting at point (XPOS,YPOS) with the current brush.

COLORINFO is not the fill color, its meaning depends on the value of FILLTYPE.

FILLTYPE can have the following values:
0 Will fill an area surrounded by border of colorinfo
1 Will fill an area whose current color is colorinfo

Polygons
If you want to draw lines in bulk or draw a polygon, you can create an array of points and use a single command to connect them or create a polygon.

For example:
    var myshape: array[1..5] of points;
    begin
      myshape[1].x:=320; myshape[1].y:=200;
      myshape[2].x:=350; myshape[2].y:=250;
      myshape[3].x:=120; myshape[3].y:=300;
      myshape[4].x:= 34; myshape[4].y:=250;
      myshape[5].x:= 1; myshape[5].y:=200;
      connectdots(myshape,5);
    end.

This code will draw lines between the points

procedure Aconnectdots(DC: SDC; var pointarray; count:integer);
procedure connectdots(var pointarray; count:integer);

This function draws lines between a group of points. POINTARRAY is the array of points, COUNT is the number of points to draw.

procedure shape(var pointarray; count,method: integer);
procedure Ashape(DC: SDC; var pointarray; count,method: integer);

This function draws a shape from the points. The shape is filled with the current brush and drawn with the current pen. POINTARRAY is the array of points, COUNT is the number of points to draw. Method can be either 1 or 2, it specifies the fill mode. The fill modes can be confusing, so often the best way to use them is to experiment. But, here was the desciption given by Microsoft in the Win32 SDK:

1 Alternate. The system fills the area between odd-numbered and even-numbered polygon sides on each scan line. That is, the system fills the area between the first and second side, between the third and fourth side, and so on.

2 Winding. The system uses the direction in which a figure was drawn to determine whether to fill an area. Each line segment in a polygon is drawn in either a clockwise or a counterclockwise direction. Whenever an imaginary line drawn from an enclosed area to the outside of a figure passes through a clockwise line segment, a count is incremented (increased by one); when the line passes through a counterclockwise line segment, the count is decremented (decreased by one). The area is filled if the count is nonzero when the line reaches the outside of the figure.

Font and Text Functions

These functions draw text on the screen. Text is plotted with the TXT command and the font is set using the SETFONT command.

procedure Asetfont(DC:SDC; fontface:string; size, weight, italic, underline, strikeout:integer; angle:real);
procedure setfont(fontface:string; size, weight, italic, underline, strikeout:integer; angle:real );

Fontface is the face of the font. It's name must be typed exactly. Examples are 'Arial' 'Times New Roman' 'Courier New' or 'Britannic Bold'

Size is the height of the font in pixels

Weight is a number from 0-9. 0 is the default weight. It can also range from values 1-9 where where 1 is thin and 9 is extra bold.

Italic,Underline,Strikeout
If these are nonzero then these traits will appear, otherwise they won't.

Angle is the angle in degrees above the horizontal to draw the font.

procedure Atxt(DC: SDC; x,y,align:integer; color:longint; text:string);
procedure txt(x,y,align:integer; color:longint; text:string);

This draws a string at position(x,y) in the specified color with the current font.

Align is a number 1-3
1 Text is left aligned at (X,Y)
2 Text is right aligned at (X,Y)
3 Text is centered at (X,Y)

function Agetlng(DC:SDC; text:string):longint;
function getlng(text:string):longint;

This function returns the width of a text string (in pixels) if it would be drawn in the current font.

Bitmap Functions
The bitmap procedures basically work by loading a bitmap from a disk into memory. Once the bitmap is in memory, it can be displayed in different ways. The commands make it look like the bitmap is being loaded into a variable, but actually the variable only holds a number value called a handle that tells Windows where the bitmap is being stored. This is only important because these bitmap variables must be treated in special ways:

1) The bitmaps MUST be deleted before the program ends. Although Turbo Pascal automatically deletes variables for you, it does not know about any bitmaps you load. Use the DELETEBMP command to do this. If you do not delete the bitmaps, they will drain system resources, and you may run out of memory.

2) Also make sure to delete the bitmap variable before you load another another bitmap into them. For example if you put:
    var MYBMP:BMP;
    begin
      MYBMP:=loadbmp('c:\russ\homer.bmp');
      MYBMP:=loadbmp('c:\russ\bomb.bmp');
    end.

The homer bitmap is still floating around in memory and using up resources. You must delete it before you load bomb.bmp.

3) You can set one bitmap variable equal to another, but even after you've done this, changing one bitmap variable will affect the other.

Example:

    var mybmp1,mybmp2:BMP;
      begin
        MYBMP1:=loadbmp('c:\russ\homer.bmp');
        MYBMP2:=MYBMP1;
        deletebmp(mybmp2);
      end.

After this, both MYBMP1 and MYBMP2 contain no bitmaps. This is because the BMP variable type is really just a handle that points to a bitmap.When you set a one BMP variable equal to another you are just pointing them toward the same bitmap. Once you delete that bitmap, neither of them will work.

function loadbmp(filename:string):BMP;

This loads a bitmap from a file into memory and returns the bitmap variable (which is really just the bitmap's handle). You must delete the bitmap with DELETEBMP when you are done with it.

procedure deletebmp(var thebmp:bmp);

This deletes a bitmap from memory. (It does NOT delete the bitmap from the screen, though. Once a bitmap has been plotted, will stay there until you draw over it with something)

procedure Adrawbmp(DC: SDC; x,y: integer; bmpname: bmp; stretched, width, height:integer);
procedure drawbmp(x,y: integer; bmpname: bmp; stretched,width,height:integer);

This draws a bitmap on the screen at point (X,Y). BMPNAME is the name
of the bitmap.

WIDTH and HEIGHT tell Windows how big you want the bitmap to be. If you set either of these to 0, the program defaults to the actual height or width of the bitmap.

If STRETCHED is nonzero then windows will stretch your bitmap to the
height and width you specify, otherwise it will crop your bitmap to the specified dimensions.

procedure Amaskbmp(TheDC:SDC; x,y: integer; themask,thepic: bmp; stretched,wth,ht:integer);
procedure maskbmp(x,y: integer; themask,thepic: bmp; stretched,wth,ht:integer);

This procedure is almost exactly the same as the drawbmp. The only difference is that you can specifiy a mask bitmap.

A mask is usually a greyscale bitmap the same size as the original bitmap. The area of the mask that are white are areas where the bitmap will cover up whatever's already on the screen. The areas that are black will leave what's already on the screen intact. Grey areas of the mask will result in a combination of whats on the screen and what's in the bitmap. Greyed and colored masks can be used for special effects. There is a demonstration of masks in the demo program.

procedure Adrawpicture(DC:SDC; x,y:integer; filename:string);
procedure drawpicture(x,y:integer; filename:string);

This command combines the LOADBMP, DRAWBMP, and DELETEBMP functions. It loads a bitmap from path in FILENAME, and draws it at (X,Y) on the screen. However, this command is a considerably slower than the other bitmap commands. It takes longer to display a bitmap from the disk than it does from memory. So this should really only be used when you are planning to display the bitmap once. This should not be used for animations.

function getwidth(thebmp:bmp):integer;
function getheight(thebmp:bmp):integer;

These functions return the height and width of a bitmap.

Information and Input Functions
function AFileOpen(HWindow:hwnd; path,ftype,wildcards: string):string;
function FileOpen(path,ftype,wildcards: string):string;

This function opens the Open common dialog box. Path is the path with the default directory, ftype is a description of the file like 'Picture File' or 'Data File', and wildcards is a string with dos wildcards showing files type to look for such as '*.txt' or '*.dat' It returns the path to a file.

function AFileSave(HWindow:hwnd; path,ftype,wildcards: string):string;
function FileSave(path,ftype,wildcards: string):string;

This function opens the Save As common dialog box. The sytax is the same as for FileOpen

function getapppath(instance: thandle): string;
function apppath: string;

These functions return the path of the program. This is useful for finding data files which are usually in the same directory as the program. Getapppath requires the instance handle of the application. This is in the variable HINSTANCE for all Pascal programs.

procedure showcursor;

This tells the EasyCRT window to display a blinking cursor in it's current position. This can be used to signify to the user that the program is accepting keyboard input. It works well with the inkeyasc function.

procedure hidecursor;

This hides the cursor for EasyCRT programs

procedure resetkeys;

This function clears the key buffer for the inkeyasc function and sets the inkey to zero. It is a good idea to include this before using the inkey or inkeyasc functions after the program has been doing an animation or something where keyboard input was not accepted.

function inkey:word;

This function works in real time. If a key is being pressed then its virtual keycode will be returned, if no key is being pressed then it will return 0. This is useful when getting input from arrow keys in a game and in some other situations. This is not good for getting text input, since there is no key buffer and it will only return a key if it is being pressed at the same time the function is called.

function inkeyasc:char;

This function is useful for getting text input. Whenever a key is pressed, the EasyCRT stores it in a buffer. This function returns the keys from this buffer in the order they were received. It will return a CHR(0) if no key has been pressed. Use the resetkeys to clear this buffer.

function isdown(virtkey: integer): boolean;

This is a way the program can check if a specific key is being pressed. Virtkey is the virtual key code of the key. The return value is boolean.

function mousex: integer;

This function returns the current x position of the mouse, relative to the EasyCRT window;

function mousey: integer;

This function returns the current y position of the mouse, relative to the EasyCRT window;

ldown, rdown: boolean

These are two public EasyCRT variables. If the left button is down then LDOWN = TRUE otherwise LDOWN = FALSE. If the right button is down then RDOWN = TRUE otherwise RDOWN = FALSE.

procedure getclick;

This pauses the program until there is a mouse click. The location of the click is stored in the point variable, LASTCLICK. LASTCLICK.X is the x position of the mouse during the click and LASTCLICK.Y is the y position of the mouse during the click.

function dchandle: thandle;

This function returns the DC handle of the EasyCRT window. This way the program can make use of GDI functions.

function windowhandle: thandle;

This function returns the window handle of the EasyCRT window. This way the program can make use of many other API window functions.

function FileExists(FileName: String): Boolean;

This function is useful for preventing File Not Found errors in a program. It tests to see if a file exists.

Program Flow Functions
procedure Aunfreeze(Wnd: Hwnd);
procedure unfreeze;

A program that is in a repeat loop is unable to process its messages. This causes it to stop responding to keypresses and mouseclicks and can cause the system to lock up. Putting an UNFREEZE in the loop forces the program to look for and respond to messages so nothing locks up.

procedure delay(milliseconds:longint);
procedure Adelay(milliseconds:longint; Wnd:Hwnd);

This command pauses the program for the specified number of milliseconds.

EasyGDI Specific Functions
function makeDC(FromWhat:Thandle; Form:Word):SDC;

This function gets a device context and initializes the data structure used to keep track of handles.

Form can have three values:

0: This creates a device context for a particular window. The window's handle is specified in FROMWHAT

1: This creates a device context in memory. A memory device context can be used to draw things off screen. It's contents can then be saved into files or projected onto areas of the screen. It can also be used to store or operate on bitmaps or do screen captures. A memory device context will not work until a bitmap has been selected into it. A bitmap with the desired size can be created with the CREATECOMPATIBLEBITMAP function and then selected into the new device context with SETBMP. This form requires the handle of a valid DC in FROMWHAT. It creates a device context which is compatible to this one.

2: This is allows the programmer to use EasyGDI functions with a DC that has already been created. The handle of this DC should go into fromwhat. Note: the KILLDC function will not delete this type of device context. It will only restore all the original pens and brushes.

procedure killDC(var it: SDC);

This procedure deletes all the pens and brushes that have been created and disposes of the device context.

procedure setbmp(DC: SDC; Picture: BMP);

This selects a bitmap into a device context. It is used with memory device contexts. (see makedc)
